# Authority-allocation / agentic-delegation topic assessment note

Use this reference during research-wiki ingests when a source compares AI assistance modes that differ mainly in **authority allocation** rather than model capability — e.g., advice/recommendation, critique/coaching, approval-gated execution, or autonomous delegation.

## Durable lesson

Do not flatten these sources into generic `human-ai-collaboration` or `ai-adoption` just because they involve humans using AI. If the source's central contribution is about who holds execution authority, whether humans can veto or modify output, whether the agent acts autonomously, or how one actor's delegation changes other actors' opportunity structures, run an explicit new-topic check for `agentic-delegation` or a related precise topic.

## Topic-map checks

- **Create/update `agentic-delegation`** when the source studies handoff of decision/action authority to an AI agent, especially under endogenous uptake, user preference for control, accountability friction, or group/team spillovers.
- **Update `human-ai-collaboration`** only for the collaboration mechanism: authority arrangement, review/control rights, calibration, and accountability.
- **Update `ai-adoption` / `ai-receptivity`** only when the source distinguishes availability/capability from actual willingness to use a delegation mode.
- **Update `automation-and-substitution`** carefully: autonomous execution in a task turn is not job-level labor substitution. State the grain of substitution.
- **Update `work-redesign`** when delegation changes the surrounding workflow or interaction environment for others, not merely the user's private productivity.
- **Update `human-ai-task-taxonomy`** when the source cleanly separates interaction modality from authority, auditability, output definition, or human persona.

## Source-record governance pitfall

If the source record has already been auto-committed with existing-topic feeds and the topic assessment then promotes a new topic, leave the added source `Feeds` link uncommitted with the topic proposal. This keeps the source commit stable while avoiding a dangling link before owner approval.
