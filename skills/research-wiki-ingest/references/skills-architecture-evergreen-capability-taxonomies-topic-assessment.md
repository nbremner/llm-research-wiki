# Skills architecture / evergreen capability taxonomies topic assessment

Use this reference when ingesting sources about 21st-century skills, future skills, skills taxonomies, competency architectures, AI-enabled skills inference, workforce capability dashboards, or evergreen capability models for AI-era work.

## Reusable topic-map pattern

Do not flatten these sources into generic `ai-workforce-impact-measurement` or `work-redesign` just because the user frames them around AI changing work. The central contribution is often upstream measurement architecture: how skills are named, bounded, validated, inferred, and aggregated before they become evidence for workforce planning.

Default homes to assess first:

- `competency-modeling` — when the source is about skill/competency labels, higher-order capability models, competency frameworks, or translating skill language into HR practice.
- `construct-validity` — when the source shows jangle fallacy, construct proliferation, semantic overlap, discriminant-validity problems, or mismatch between labels and measures.
- `human-capital-resource-measurement` — when individual skills or competencies are aggregated into unit/firm capability, workforce dashboards, capability-gap analysis, or human-capital claims.
- `job-analysis` — when the source ties future skills to changing work behavior, KSAOs, task context, or the need to keep broad skill labels job-relevant.
- `complex-collaborative-problem-solving` — when the source specifically discusses CPS/ColPS, transversal/nonroutine skills, dynamic assessment tasks, microworlds, or behavior-rich assessment of future-relevant capability.
- `ai-workforce-impact-measurement` — only when the source supplies measurement of AI capability/use/exposure/outcomes, or explicitly connects skills architecture to realized AI workforce impact. Do not use it as a generic dump for every AI-era skill taxonomy.

## Promote / defer guidance

Promote a standalone topic such as `skills-architecture` or `evergreen-capability-taxonomies` only when multiple sources support the architecture itself as a recurring construct: e.g., validation evidence for skills taxonomies, fairness/employee-experience evidence around skills inference, taxonomic comparisons, outcome evidence for workforce planning, or clear design principles for skill architecture under AI-driven work change.

Defer when the source mainly adds a construct-validity warning or one implementation case. In that case, update the existing topics above and keep/expand the overview watchlist item rather than creating a thin new page.

## Typical synthesis frame

The durable point is not “future skills matter.” The stronger synthesis is: AI-era workforce planning needs stable capability categories, but those categories become dangerous if they are built from fashionable overlapping labels. Skill architectures should be pruned for semantic overlap, discriminant validity, and job-relevant behavioral evidence before organizations infer, aggregate, or dashboard them as human capital.
