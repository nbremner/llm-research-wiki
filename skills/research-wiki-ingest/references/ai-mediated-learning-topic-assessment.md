# AI-mediated learning topic assessment

Use this reference when ingesting sources where AI assistance affects learning, skill formation, cognitive debt, debugging ability, comprehension, transfer, or expertise development.

## Reusable topic-map pattern

Do not flatten these sources automatically into `ai-induced-skill-erosion`, `automation-complacency`, or `critical-thinking`.

- `ai-induced-skill-erosion` is about durable loss of practiced capability, judgment, craft, confidence, or professional identity over time.
- `automation-complacency` is about reduced attention, effort, scrutiny, or oversight in the moment.
- `critical-thinking` is about verification, stewardship, judgment, and task-relevant reasoning.
- `ai-mediated-learning` is the focal home when the source asks whether AI-supported work episodes build durable competence or bypass the practice, error-resolution, explanation, recall, debugging, and transfer operations that learning requires.

## Promotion signal

Promote or update `ai-mediated-learning` when a source contributes one of these distinct claims:

1. AI access changes post-task learning, comprehension, recall, transfer, or skill evaluation, even if immediate output quality or speed is acceptable.
2. Interaction patterns distinguish productive cognitive effort from harmful bypass — e.g., conceptual inquiry, explanation-seeking, comprehension checks, generation-then-understanding, full delegation, progressive reliance, or AI-assisted debugging.
3. The source separates immediate performance from competence formation, especially for novice workers, junior professionals, onboarding, or safety-critical supervision.
4. The source provides evidence about AI-mediated learning design: learning modes, cognitive forcing, critique, pair-programming-style assistance, tutoring, feedback, or assessment of transfer.

## Good synthesis move

The core distinction to preserve is:

> AI can help produce the artifact while weakening, preserving, or strengthening the learning episode that determines future oversight capacity.

When the user explicitly asks for **cognitive debt + learning transfer + individual differences**, do not treat individual differences as a generic moderator paragraph. Map the source across:

- `ai-mediated-learning` for transfer, comprehension, effort-preserving scaffolding, and whether AI use builds durable capability.
- `cognitive-load-in-ai-assisted-work` for the productive-effort vs extraneous-burden distinction, especially pacing, notifications, context switching, interface burden, and load-sensitive scaffolds.
- `ai-induced-skill-erosion` only as a longer-run accumulation risk or precursor; avoid implying longitudinal erosion unless measured.
- `digital-work-strain` when cognitive debt includes recovery failure, burnout, attentional fragmentation, silent overload, or output metrics hiding sustainability costs.
- `inclusive-hr-systems` when the paper argues that AI workflow assumptions encode neurotypical pacing, sensory tolerance, interaction norms, or availability expectations. Frame neurodivergent workers as design SMEs / evidence about the work system when the source supports that move, not merely as a vulnerable subgroup.

When threading into existing topics, keep the specific role clear:

- In `ai-induced-skill-erosion`, frame AI-mediated learning failures as possible immediate precursors to longer-run capability loss, not proof of longitudinal erosion unless the design actually measures it.
- In `automation-complacency`, frame bypassed struggle/error-resolution as a learning-centered complacency mechanism, not simply advice-taking.
- In `critical-thinking`, frame explanation-seeking, debugging, and comprehension checks as stewardship behaviors that also build competence.
- In `cognitive-load-in-ai-assisted-work`, distinguish productive cognitive effort from extraneous interaction burden; reducing all effort can remove the friction needed for learning.
- In `human-ai-collaboration`, ask whether the episode is meant to produce a deliverable, build competence, or both.

## Boundary language

Avoid overclaiming short studies as durable skill erosion. Use language like:

- "short-horizon causal evidence of impaired skill formation"
- "plausible precursor to longer-run skill erosion"
- "learning risk before workplace capability loss is visible"
- "mechanism evidence rather than longitudinal professional-development evidence"

## Example anchor

`2026-shen-ai-impacts-skill-formation` promoted `ai-mediated-learning` because it showed that AI access in a randomized software-learning task reduced post-task skill evaluation scores by 17% without significantly improving completion time, and because interaction patterns separated low-learning delegation/debugging reliance from higher-learning conceptual inquiry and explanation-oriented use.
